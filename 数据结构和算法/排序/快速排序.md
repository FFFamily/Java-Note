# 快速排序

快速排序（Quicksort）是对冒泡排序的一种改进



基本思想

通过一趟排序将要排序的数据分割成独立的两部分，其中一部分的所有数据都比另外一部分的所有数据都要小，然后再按此方法对这两部分数据分别进行快速排序，整个排序过程可以递归进行，以此达到整个数据变成有序序列



代码

```java
 public static void quickSort(int[] arr,int left, int right) {
        int l = left; //左下标
        int r = right; //右下标

        //pivot 中间的那个值
        int pivot = arr[(left + right) / 2];
        //临时变量，作为交换时使用
        int temp = 0;
        //while循环的目的是让比pivot 值小放到左边,比pivot 值大放到右边
        //当左下标小于右下标时才排序，不然就会越过中间值
        while (l < r){
            /**
             * 两个while设计为了找到左右两边各不满足条件的值的位置
             */
            while (arr[l]<pivot){
                //当左边的值小于中间值时，就移动到下一个继续判断
                l++;
            }
            while (arr[r]>pivot){
                //当右边的值大于中间值时，就移动到下一个继续判断
                r--;//减的原因是右边是从后开始的
            }
            //结束标志
            if (l >= r){
                //如果l >= r说明pivot 的左右两的值，已经按照左边全部是小于等于pivot值，右边全部是大于等于pivot值
                //这个时候就需要跳出这个循环
                break;
            }


            /**
             * 问题
             * 例子：【1,3,4,8,9,5,10】
             * 如果我左边遍历完了都比中值小
             * 但是右边却有个比中值小的
             * 按道理这个时候 l 是对应中值的
             * 那么右边和左边一交换不就乱了吗
             * 解决
             * 考虑到这种情况，就有了后面两步对pivot值的判断，右边等于就让左下标+1.左边等于就让右下标-1
             * 但是
             * 不进行判断也没有影响结果，这是为什么呢？
             * 解决
             * 针对小的数组不交换没有影响，但是数组一旦大了起来就会出错，StackOverflowError
             * 但是
             * 为什么会出现栈溢出呢？
             * 解决
             * 通过实验，发现不是数组大小会产生栈溢出,80000的数据也没有产生栈溢出，猜测是哪种特殊的情况
             */
            //如果发现有不满足大小匹配的，就需要交换
            temp = arr[l];
            arr[l] = arr[r];
            arr[r] = temp;

            //如果交换完后，发现这个arr[l] == pivot值 相等 r--， 前移
            if(arr[l] == pivot) {
                r -= 1;
            }
            //如果交换完后，发现这个arr[r] == pivot值 相等 l++， 后移
            if(arr[r] == pivot) {
                l += 1;
            }

            // 如果 l == r, 必须l++, r--, 否则为出现栈溢出
            /**
             * 问题：什么时候就会出现 l == r
             */
            if (l == r) {
                l += 1;
                r -= 1;
            }

            //向左递归
            if(left < r) {
                quickSort(arr, left, r);
            }
            //向右递归
            if(right > l) {
                quickSort(arr, l, right);
            }
        }
```
